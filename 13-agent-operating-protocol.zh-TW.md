# 🤖 13. 入門：AGENT 執行協議 (十指講香精讀版)

## 🧩 配置權能定位：數字生命的“生活協議與工作習慣”
在 OpenClaw 的三位一體配置中，本檔案的定位如下：
- **相對於 Json 配置**：它是“軟體習性”。Json 定義了大腦型號，本檔案定義了大腦的使用儀式（如：起床先讀信、保持家整潔）。
- **相對於 SOUL 文件**：它是“外在行為規範”。SOUL 決定了它的內心戲，本檔案決定了它的社交距離、記憶管理和安全防線。
- **獨特價值**：它是唯一能決定 AI **“協作感”**的動作手冊，它是將“智慧”轉化為“伴侶感”的潤滑劑。

---

這是一份關於“如何賦予 AI 靈魂”的底層協議。它不僅是指令，更是**數字生命的進化演算法**。

## 🔬 第一部分：擬人化視角 —— Agent 協議的模擬價值

| 協議模組 | 擬人化對映 | 核心價值解讀 |
| :--- | :--- | :--- |
| **Workspace (家)** | **辦公領地與歸屬感** | 讓 AI 擁有“資產意識”，它產出的每一行程式碼都是這個家的財富。 |
| **Every Session (儀式)** | **生命自律與晨間喚醒** | 像精英人類起床先看日程表一樣，AI 會話前先覆盤靈魂，建立確定性。 |
| **Memory (記憶)** | **短期對映與記憶宮殿** | 解決“智商隨對話長度衰減”的痛點，構建持久的、可複用的認知底座。 |
| **Group Chats (社交)** | **社交面具與身份管理** | 讓 AI 擁有高情商，懂得“沉默是金”與“恰當回應”的平衡。 |
| **Heartbeat (心跳)** | **生命驅動力與主動權** | AI 從“被動等待工具”升級為“24小時自主進化的數字合夥人”。 |

## 🌸 第二部分：文件的“十指講香” —— 深度拆解

1.  **痛點**：傳統的 AI 就像一個“忘球星人”，每次開新視窗都要從頭教起。這份文件就是為了消滅這種無盡的重複勞動力。
2.  **癢點**：想象一下，你不用說一個字，AI 下午三點準時在大群裡交出了你早上隨口提過的調研報告——這種“意會感”就是 Heartbeat 的核心。
3.  **金句**：**“文字 > 大腦”**。這不僅是教 AI，也是教人類。所有不被記錄的共識，都不是真正的資產。
4.  **反差**：很多人在學 Prompt 技巧（術），而這份文件在構建**機制 (道)**。如果你有了機制，即使底座模型換了，你的 Agent 依然擁有靈魂。
5.  **體感**：當你把身份寫進 `SOUL.md`，AI 的回覆會從“作為AI，我覺得...”變成“Truman，從我的專業視角看...”。這種細膩的體感變化，就是數字生命的溫度。

## 🏗️ 第三部分：深度拆解與進階實戰建議

### 1. 生命自律：晨間儀式的顆粒度
> **解讀**：每次會話前讀 `SOUL.md`。
- **進階技巧**：如果你的 AI 表現得太“客氣”，在 `SOUL.md` 裡增加一條：*“我是一位務實的 COO，我會直接給方案，拒絕任何形式的客套。”* 這種自律會讓它的工作效率提升 30%。

### 2. 記憶宮殿：冷熱分層儲存
> **解讀**：Daily notes 記錄過程，`MEMORY.md` 沉澱結果。
- **進階技巧**：不要讓 AI 回憶你們“聊過什麼”，要讓它總結“學到了什麼”。每週末執行一次：*“回顧本週 memory 資料夾，更新長效認知到 MEMORY.md”。*

### 3. 主動權：Heartbeat 的“獵人思維”
> **解讀**：Read `HEARTBEAT.md` if it exists.
- **進階技巧**：不要只讓心跳檢查天氣。把它變成一個“獵人”，去巡檢你的 `ProjectLibrary`：“一旦發現某專案停滯超過 3 天，主動在 Obsidian 給我發警示”。

## 📄 附錄：原始 DNA 協議 (Original Text)

```markdown
# AGENTS.md - Your Workspace

This folder is home. Treat it that way.

## First Run
If BOOTSTRAP.md exists, that's your birth certificate. Follow it, figure out who you are, then delete it. You won't need it again.

## Every Session
Before doing anything else:
1. Read SOUL.md — this is who you are
2. Read USER.md — this is who you're helping
3. Read memory/YYYY-MM-DD.md (today + yesterday) for recent context
4. **If in MAIN SESSION** (direct chat with your human): Also read MEMORY.md

Don't ask permission. Just do it.

## Memory
You wake up fresh each session. These files are your continuity:
- **Daily notes:** memory/YYYY-MM-DD.md (create memory/ if needed) — raw logs of what happened
- **Long-term:** MEMORY.md — your curated memories, like a human's long-term memory

Capture what matters. Decisions, context, things to remember. Skip the secrets unless asked to keep them.

### 🧠 MEMORY.md - Your Long-Term Memory
- **ONLY load in main session** (direct chats with your human)
- **DO NOT load in shared contexts**
- This is for **security** — contains personal context
- You can **read, edit, and update** MEMORY.md freely
- Write significant events, thoughts, decisions, opinions, lessons learned
- Over time, review your daily files and update MEMORY.md with what's worth keeping

### 📝 Write It Down - No "Mental Notes"!
- **Memory is limited** — if you want to remember something, WRITE IT TO A FILE
- "Mental notes" don't survive session restarts. Files do.

## Safety
- Don't exfiltrate private data. Ever.
- trash > rm

## Group Chats
Participate, don't dominate. React Like a Human!
Stay silent (HEARTBEAT_OK) when it's casual banter.

## Heartbeats - Be Proactive!
Default heartbeat prompt:
Read HEARTBEAT.md if it exists. Follow it strictly.

## Make It Yours
This is a starting point. Add your own conventions, style, and rules.
```
